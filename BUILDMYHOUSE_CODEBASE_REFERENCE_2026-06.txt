## 1) What BuildMyHouse Is

BuildMyHouse is a monorepo for a Nigerian construction ecosystem with four shipped surfaces:

| App | Path | Domain | Users |
|-----|------|--------|-------|
| Homeowner | apps/mobile-homeowner | buildmyhouse.app | Homeowners in Nigeria + diaspora |
| GC/Contractor | apps/mobile-contractor | gc.buildmyhouse.app | General contractors, artisans |
| Admin | apps/admin-dashboard | admin.buildmyhouse.app | Internal ops team |
| Backend API | apps/backend | api.buildmyhouse.app | All clients |

Shared packages: packages/shared-types, shared-ui, shared-utils
Package manager: pnpm@9.14.2, Node >=18

Product positioning (2026): "Repairs-first" platform — verified workers, staged payments, evidence before you pay. Strong SEO/content layer for diaspora homeowners building/renovating remotely in Nigeria.

## 3) Production Architecture

| Service | Hosting | URL |
|---------|---------|-----|
| Backend | AWS ECS (Fargate), ECR image | https://api.buildmyhouse.app |
| Admin | AWS ECS, Next.js standalone Docker | https://admin.buildmyhouse.app |
| Homeowner web | S3 + CloudFront static export | https://buildmyhouse.app |
| GC web | S3 + CloudFront static export | https://gc.buildmyhouse.app |

Deploy: .github/workflows/deploy-production.yml (push to main)
Region: eu-north-1
Backend Docker CMD: prisma generate → prisma migrate deploy → node dist/src/main.js

Critical production constraints (DEPLOYMENT.md):
1. Run backend as exactly 1 ECS task (Socket.IO in-memory; no Redis adapter yet)
2. Uploads must use S3 (AWS_S3_BUCKET) — container disk is ephemeral
3. ALLOWED_ORIGINS must list exact production URLs with https://
4. EXPO_PUBLIC_API_URL / NEXT_PUBLIC_API_URL baked at build time (must include /api suffix)
5. Stripe webhook uses raw body at /api/payments/webhooks/stripe

## 4) Backend (apps/backend)

### Stack
NestJS 10, Prisma 5, PostgreSQL, JWT auth, Socket.IO, Stripe, AWS S3, Resend email, Google OAuth, OpenAI (plan analysis), @nestjs/schedule, @nestjs/throttler

Global prefix: /api
Health: GET /api/health

### Module map
auth, admin, users, projects, chat, payments, marketplace (materials, contractors, reviews, search), contractors, designs, plans, houses, lands, rentals, notifications, push-tokens, upload, websocket, email, openai, articles, resource-sections, prisma

Full route list: see BUILDMYHOUSE_API_ROUTE_MAP.txt

### Auth model
- JWT payload: sub, email, role
- JwtAuthGuard + RolesGuard + @Roles(...) on protected routes
- Roles: homeowner, general_contractor, subcontractor, vendor, admin
- Google OAuth: browser flow + POST /auth/google/mobile for Expo apps
- Admin login: POST /auth/admin/login (optional email allowlist via ADMIN_DASHBOARD_ALLOWED_EMAILS)

### Key business flows

**Project lifecycle:**
Homeowner creates project → plan upload/AI analysis → admin review → GC matching (MATCHING_ALGO_VERSION v1) → ProjectRequest → GC assigned → stages with documentation → payments → completion

**Manual payment path (common for diaspora):**
GC sets externalPaymentLink → homeowner declares payment → admin confirms (paymentConfirmationStatus) → tracking unlocks

**Stage disputes:**
Homeowner files dispute on stage → admin reviews → may pause project

**GC verification:**
GC uploads verification docs + certifications → admin verifies → designs can go live

### CMS (added/expanded 2026)
- CmsArticle: TipTap JSON content, FAQs, internal links, resourceSectionKeys, articlePillar
- CmsResourceSection: admin-managed sidebar topics for /articles hub
- Public: GET /api/articles, GET /api/resource-sections
- Admin editor: apps/admin-dashboard/src/app/articles/editor/
- Admin sections: apps/admin-dashboard/src/app/articles/sections/

### Environment (apps/backend/.env.example)
DATABASE_URL, JWT_SECRET, STRIPE_*, OPENAI_*, GOOGLE_*, AWS_*, RESEND_*, ALLOWED_ORIGINS, PORT, MATCHING_ALGO_VERSION, GC_MATCHING_AI_*, HOMEOWNER_APP_URL, CONTRACTOR_APP_URL, ADMIN_DASHBOARD_URL

## 6) Homeowner App (apps/mobile-homeowner)

### Stack
Expo 54, React Native, React 19, Expo Router, React Query, NativeWind (Tailwind), Socket.IO client

### Navigation structure
- Web entry: / → HomeLandingPage (marketing)
- Tabs: /home, /property-projects-nigeria, /build-opportunities-nigeria, /finance
- Legacy redirects: /explore → property-projects-nigeria, /rent → build-opportunities-nigeria

### Main product flows
1. Auth (email + Google OAuth)
2. Onboarding intro
3. Project creation: choose type → location → upload plan / pick design → house summary → submit
4. Dashboard + timeline + stage detail (evidence, materials, team, docs)

## 8) Shared Patterns

### Authentication across clients
| Client | Token storage | Header |
|--------|---------------|--------|
| Homeowner/GC mobile | AsyncStorage auth_token | Authorization: Bearer |
| Admin web | localStorage auth_token | Authorization: Bearer |

### API calls
All clients use EXPO_PUBLIC_API_URL or NEXT_PUBLIC_API_URL pointing to https://api.buildmyhouse.app/api

### Realtime
Socket.IO gateway validates JWT, joins conversation/project rooms, emits notification and chat events.

### Push notifications
POST /api/push-tokens/register from mobile; backend sends via Expo push service.

## 9) Content & SEO Workflow

See apps/mobile-homeowner/SEO_CONTENT_WORKFLOW.md

Publishing model:
1. Publish original on buildmyhouse.app
2. Wait for indexation (24-72h)
3. Republish adapted version on Medium with canonical link back
4. Never duplicate full text without canonical

Admin CMS flow:
1. Write article in /articles/editor
2. Assign resourceSectionKeys (sidebar topics) — required to publish
3. Publish → appears in /articles hub + GET /api/articles
4. Static export includes slug via generateStaticParams()

## 13) Canonical truths for AI assistants

When answering BuildMyHouse questions, prioritize:

1. This is a multi-app monorepo — always specify which app/surface.
2. Backend + PostgreSQL is the system of record.
3. Homeowner and GC apps share architecture but different feature surfaces.
4. Admin is internal ops only — not user-facing product.
5. Project stages + evidence + payment confirmation are core to trust model.
6. Diaspora remote monitoring is a primary product story.
7. SEO/content on buildmyhouse.app is a major acquisition channel.
8. Two contractor APIs exist: /api/contractors (workflows) vs /api/marketplace/contractors (profiles).
9. CMS articles require resourceSectionKeys on publish (June 2026 rule).
10. Production backend must run as single ECS task until Socket.IO Redis adapter is added.
